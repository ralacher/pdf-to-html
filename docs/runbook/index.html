Section 01

System Architecture

The NCDIT WCAG Document Converter is a serverless document processing pipeline built on Azure Functions. Files uploaded through the Next.js frontend are stored in Azure Blob Storage, triggering automated conversion workflows that extract content, apply WCAG compliance rules, and generate accessible HTML output.

Section 02

Deployment Guide

Prerequisites

  • Azure subscription with Contributor access
  • Azure CLI installed and authenticated
  • Python 3.12+, Node.js 20+
  • Azure Functions Core Tools v4

Backend Deployment

Backend Setup 
# Login to Azure
$ az login

# Create resource group
$ az group create --name rg-ncdit-wcag --location eastus2

# Create storage account
$ az storage account create --name ncditwcagstorage \
  --resource-group rg-ncdit-wcag --sku Standard_LRS

# Create function app
$ az functionapp create --name ncdit-wcag-converter \
  --resource-group rg-ncdit-wcag \
  --storage-account ncditwcagstorage \
  --runtime python --runtime-version 3.12 \
  --functions-version 4 --os-type Linux

# Deploy
$ func azure functionapp publish ncdit-wcag-converter

Frontend Deployment

Frontend Setup 
$ cd frontend
$ npm run build

# Deploy to Azure Static Web Apps
$ az staticwebapp create --name ncdit-wcag-ui \
  --resource-group rg-ncdit-wcag \
  --source . --location eastus2
✅

Deployment Complete: Verify the function app is running in Azure Portal → Function Apps → ncdit-wcag-converter → Functions. Check that the frontend is accessible at the Static Web App URL.

Section 03

Configuration Reference

All configuration values are stored as Application Settings in the Azure Function App. Set these via Azure Portal or Azure CLI.

Variable Required Default Description
AzureWebJobsStorage ✅ — Azure Storage connection string
AZURE_DI_ENDPOINT ✅ — Document Intelligence endpoint URL
AZURE_DI_KEY ✅ — Document Intelligence API key
UPLOAD_CONTAINER ❌ files Input blob container name
OUTPUT_CONTAINER ❌ output Output blob container name
FUNCTIONS_WORKER_RUNTIME ✅ python Must be "python"
MAX_FILE_SIZE_MB ❌ 100 Maximum upload file size
⚠️

Secrets Management: Never commit AZURE_DI_KEY or connection strings to source control. Use Azure Key Vault references for production deployments.

Section 05

Operations & Monitoring

Health Checks

  • Azure Functions: Azure Portal → Function App → Functions → Monitor
  • Storage: Azure Portal → Storage Account → Monitoring → Metrics
  • Frontend: Check deployment URL returns 200

Key Metrics to Monitor

  • Function execution count and duration
  • Blob storage operations (reads/writes)
  • OCR API call count and latency
  • Error rate (failed conversions / total)
  • Queue depth (pending documents)

Log Access

Log Streaming 
# Stream live function logs
$ func azure functionapp logstream ncdit-wcag-converter

# Query Application Insights (if configured)
$ az monitor app-insights query --app ncdit-wcag-insights \
  --analytics-query "traces | where timestamp > ago(1h) | order by timestamp desc"
ℹ️

Application Insights: Enable Application Insights in the Function App for advanced monitoring, distributed tracing, and custom telemetry. Configure alerts for high error rates or latency spikes.

Section 06

Troubleshooting

Common issues and their resolutions. Check logs and metrics first for additional context.

Symptom Likely Cause Resolution
Blob trigger not firing Storage connection string wrong Verify AzureWebJobsStorage in app settings
OCR returns empty results Document Intelligence quota exhausted Check Azure portal for quota usage, upgrade tier
WCAG violations in output Heading hierarchy broken Check html_builder.py heading enforcement
Upload fails at 0% SAS token expired or CORS misconfigured Regenerate token, check CORS on storage
"Password-protected" error Encrypted document uploaded User must remove password before uploading
Conversion timeout (>5min) Large scanned PDF, OCR slow Check DI tier, consider S0 → S1 upgrade
Frontend build fails Node version mismatch Ensure Node.js 20+ installed

Section 07

Incident Response

Severity Levels

P1 — Critical (Service Down)

  1. Check Azure Functions health in portal
  2. Check storage account connectivity
  3. Review last deployment (rollback if needed): func azure functionapp publish --slot staging
  4. Escalate to Azure support if infrastructure issue
🔴

P1 Alert: Notify on-call engineer immediately. Document all troubleshooting steps in incident ticket. Target resolution time: 1 hour.

P2 — High (Degraded Performance)

  1. Check function execution duration metrics
  2. Review OCR API latency
  3. Check for throttling (429 responses)
  4. Scale up Function App plan if needed

P3 — Medium (Individual Failures)

  1. Check document-specific error in blob metadata
  2. Retry failed document: re-upload to trigger reprocessing
  3. Check if document is corrupted or password-protected

Escalation Contacts

Role Contact When
On-call Engineer (team distribution list) All P1/P2
Azure Support support.microsoft.com Infrastructure issues
NCDIT Project Lead (internal) Policy/compliance questions
Section 08

Security & Compliance

Compliance Overview

  • WCAG 2.1 AA Compliance: All HTML output validated against 7 rules
  • File Validation: Only PDF, DOCX, PPTX accepted (max 100MB)
  • Encrypted Documents: Automatically rejected with user-friendly error
  • SAS Tokens: Short-lived (1 hour), scoped to specific blob
  • No PII in Logs: Document content never logged
  • Key Rotation: Rotate DI keys quarterly via Azure Portal
  • Access Control: Function keys for API, Azure RBAC for resources
  • Data Residency: All processing in Azure East US 2

Key Rotation Procedure

Key Rotation 
# Rotate Document Intelligence keys
$ az cognitiveservices account keys regenerate \
  --name ncdit-doc-intelligence \
  --resource-group rg-ncdit-wcag --key-name key1

# Update function app setting
$ az functionapp config appsettings set \
  --name ncdit-wcag-converter \
  --resource-group rg-ncdit-wcag \
  --settings "AZURE_DI_KEY=<new-key>"
⚠️

Security Reminder: Rotate all keys quarterly. Use Azure Key Vault for production secrets. Enable soft delete on blob storage (14-day retention) to recover accidentally deleted files.

  • Configuration: Exported via az functionapp config appsettings list
