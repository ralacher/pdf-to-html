## Key Decisions

1. **PDF.js for browser-based rendering** — Server-side PDF rendering would require heavy dependencies. PDF.js is lightweight, handles viewer state client-side, and avoids additional backend load.
2. **SAS token reuse from upload flow** — Download SAS tokens follow the same generation pattern as upload tokens, leveraging existing `statusService` infrastructure.
3. **Modal for preview** — Non-modal preview would require router state management; modal keeps the dashboard UX simple and focused.
